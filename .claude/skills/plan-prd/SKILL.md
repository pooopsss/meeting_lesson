# Plan Generator

Прочитай PRD из файла: $ARGUMENTS

Создай план реализации и сохрани в файле 'docs/plan-$ARGUMENTS.md' (используй перевод на английский и kebab-case)

## Структура плана

#Plan: {название фичи}
**PRD:** $ARGUMENTS
**Дата:** {текущая дата}

## Фазы реализации

### Фаза 1: {название}
**Цель:** Что дает эта фаза
**Затрагивает:** backend / frontend / database
**Задачи:**
- [ ] Задача 1
- [ ] Задача 2
**Когда готова:** Конкретный критерий

### Фаза 2: {название}
...


## Правила разбивки на фазы:
- Каждая фаза должна давать рабочий результат
- Фазы независимы, можно остановиться после любой.
- Первая фаза как минимальный рабочий путь (Tracer Bullet).
- Не больше 5 задач в одной фазе
- frontend и backend одной фичи в разные фазы
- В каждой фазе должны быть запланированные тесты для покрытия функционала фазы.

## Правила
- Читай PRD внимательно, план должен покрывать все критерии готовности.
- Не добавляй задачи которых нет в PRD. Если PRD неполный, задай вопрос перед созданием плана
- Основной язык - Русский
